Kanban Board

The Kanban is your central order management view. Move orders through their lifecycle, record payments, and dispatch for delivery.

When to use: Throughout the day to track order status, confirm incoming orders, record payment receipts, and dispatch delivery trips.

Order Status Columns

📥 Recieved / Received

New order just submitted from POS. The app may show the first column as Recieved.

🧑‍🍳 In Progress

Order accepted and currently being prepared.

✅ Ready

Order is prepared and ready for pickup or delivery dispatch.

🚚 Out for Delivery

Order dispatched with a courier. Delivery in progress.

🎉 Delivered

Order delivered and payment settled.

⛔ Cancelled

Cancelled invoices appear here for visibility and follow-up.

Step-by-step: Daily Kanban Workflow

1

Open the Kanban Board

Tap Kanban from the side menu or the bottom navigation. All orders for your branch appear in their respective columns.

2

Accept a new order

Find the order in the Recieved / Received column. Tap the order card, then tap Accept Order if the button is shown. Move or update the order to In Progress when preparation starts.

3

Move the order to Ready when prepared

When production is finished, move or update the order to Ready. Ready means the order can now be picked up by the customer or assigned to a delivery trip.

4

Record a payment receipt if needed

On an order card, tap More Options (⋮) → Payment Receipts. Add a new receipt by entering the amount and payment method. Receipts await manager confirmation before the order is considered fully paid.

5

Move Ready orders to Out for Delivery

When a delivery order is ready to dispatch, tap the order card → tap the Delivery action button. A dialog asks you to confirm payment status and assign a courier. If the order is unpaid, choose the correct courier collection option before proceeding.

If the order is going into a Delivery Trip, use the Trips screen instead.

6

Delivered and Cancelled orders

After delivery is confirmed, the order appears in Delivered. Orders cancelled by an authorized user appear in Cancelled. Cancelled is not a normal delivery step; it is a visibility column for orders that were stopped.

Filters & Search

Show Filters: Tap the filter icon to filter by customer name, status, date range, or amount range.

Search orders: Use the search bar to find an order by customer name or invoice number.

Filter by Branch: If you have access to multiple branches (unusual for staff), use the branch filter dropdown.

Refresh Orders: Tap the refresh icon to pull the latest orders from the server.

Editing an Invoice (Amendment)

If you need to correct a submitted order (e.g. wrong item, wrong quantity), tap the order card → More Options (⋮) → Edit Invoice. This opens an amendment draft in the POS. Make corrections, then tap Submit Amendment. The original invoice is replaced.

The Edit Invoice option only appears on invoices that can still be amended (i.e. not yet Out for Delivery, Delivered, or Cancelled).

💡 Tips

  • Check the Kanban at the start of every shift to see if any overnight orders are still in Recieved / Received.
  • An order with a red "UNPAID" badge must have payment collected before marking Out for Delivery.
  • Use Change Delivery Slot (via More Options) if the customer requests a different time after ordering.

⚠️ Common Issues

  • Order does not move to next column: The server action may have failed. Check your internet connection and try again.
  • "Invoice is UNPAID" warning: Collect payment from the customer or choose the courier-collects option before dispatching.
  • Cannot find an order: Use the search bar or clear all filters. Orders older than the current date range may be filtered out.
